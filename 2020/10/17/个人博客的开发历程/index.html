说明：采用hexo+github的方式搭建，域名地址为：raindropstad.top

开发历程

hexo+github搭建个人网页

搭建环境

参考链接：

  • 使用hexo+github免费搭建个人博客网站超详细教程
  • 使用hexo+github搭建免费个人博客详细教程
  • 修改Nodejs内置的npm默认配置路径

采用版本说明

  • windows10
  • node.js v14.13.1
  • git version 2.21.0.windows.1
  • hexo-cli: 4.2.0

  • 注意：node.js的安装最好按默认的位置，这样不需要重新配置系统环境变量，如果选择另外的位置，则需要配置一下path变量，指定到node的位置即可

  • 注意：cmd的打开以及git bash的打开一定一定要是管理员模式打开，不然后续操作中可能会跳出权限不足的bug

安装好node后，需要配置npm的环境

  1. 查看npm的配置环境

  2. 修改cache和prefix的路径

    如果nodejs是按默认路径配置，则位置在C:\Program Files\nodejs

    如果修改了，自己找找了hxd

    进入npm的目录下，默认路径是在C:\Program Files\nodejs\node_modules\npm

    新增两个目录 node_cache、node_global_modules

    打开cmd设置cache和prefix的路径

    依次执行：

    npm config set prefix=” C:\Program Files\nodejs\node_modules\npm\node_global_modules “

    npm config set cache=” C:\Program Files\nodejs\node_modules\npm\node_cache “

  3. 修改系统环境变量

    新增系统环境变量 NODE_HOME 指定nodejs的位置

    在PATH中点击编辑新增四条

  4. 至此，npm的路径配置完成，可在cmd中输入命令npm config ls查看路径

配置好npm后，则可以开始hexo的下载了（以下命令在git base中完成）

  1. 安装hexo 默认下载在npm的下载路径

  2. 初始化，可以在D盘新增一个工作路径

    注意，hexo init 运行后可能会报如下错误

    权限不够，忘记具体的错误提示了，注意有无权限单词字样，选择管理员打开即可

Failed to install dependencies. Please run ‘npm install’ manually!

按提示输入 npm install 后依旧报错

错误提示中包含npm ERR! Failed at the ejs@2.7.4 postinstall script.

根据提示是缺少 ejs@2.7.4 postinstall script

这是执行相对应的下载就可以了

缺什么就下载什么！

  1. 至此，hexo已经成功了一大半了，输入hexo g加载基础文件

  2. 运行

    运行结果：如果有错误，请检测是否在hexo init时有错误

    至此，可以打开localhost:4000 查看界面了 ctrl + c结束

后续操作需要

  1. 下载主题
  2. 在github上新建 用户名.github.io的项目 （需要更改网络的dns地址为 223.5.5.5）
  3. 配置密钥
  4. 在git base中下载hexo的上传工具
  5. 将github上的项目克隆下来后 上传本地仓库
  6. 配置域名（推荐腾讯云，腾讯云的客服是小姐姐，阿里云是粗汉）域名解析在腾讯云和阿里云中实现即可

后续操作相对简单，参照上面的参考链接即可（我懒了）
